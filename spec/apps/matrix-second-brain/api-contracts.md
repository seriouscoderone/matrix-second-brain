# Matrix Second Brain -- App-Level API Contracts

## API Conventions

This application has **no HTTP API**. All interactions are mediated through the Matrix protocol (event-driven) and internal TypeScript service functions. This document defines:

1. **Matrix Event Contracts** -- the inbound events the bot processes and the outbound messages it sends
2. **Service Layer Contracts** -- the internal TypeScript function signatures that form the application's logical API
3. **Database Query Contracts** -- the typed query functions that form the data access layer
4. **LLM I/O Contracts** -- the schemas for LLM input prompts and validated output

- **Style**: Event-driven service (Matrix Client-Server protocol)
- **Authentication**: Matrix access token (bot), Matrix homeserver auth (users)
- **Date Format**: ISO 8601 for LLM I/O; JavaScript `Date` objects internally
- **ID Format**: UUID v4 (auto-generated by PostgreSQL `gen_random_uuid()`)
- **Error Handling**: Try/catch with thread-reply error messages to the user; `console.error` for logging

## Inbound Event Contracts

### Text Message (m.text)

**Trigger**: User sends a text message in any room the bot is a member of.

```typescript
// Matrix event shape (relevant fields)
interface InboundTextEvent {
  type: 'm.room.message';
  event_id: string;            // e.g., "$abc123"
  sender: string;              // e.g., "@alice:localhost"
  origin_server_ts: number;    // Unix ms timestamp
  room_id: string;             // e.g., "!roomId:localhost"
  content: {
    msgtype: 'm.text';
    body: string;              // The raw message text
  };
}
```

**Routing**: See `message-routing.feature.md` for full routing logic.

### Location Event (m.location)

**Trigger**: User shares a live location or one-time location.

```typescript
interface InboundLocationEvent {
  type: 'm.room.message';
  event_id: string;
  sender: string;
  origin_server_ts: number;
  room_id: string;
  content: {
    msgtype: 'm.location';
    body: string;
    geo_uri?: string;                         // "geo:37.7749,-122.4194"
    'org.matrix.msc3488.location'?: {         // MSC3488 fallback
      uri: string;                            // "geo:37.7749,-122.4194"
    };
  };
}
```

**Parsing**: `geo_uri` is preferred; `org.matrix.msc3488.location.uri` is used as fallback. Regex: `/^geo:([-\d.]+),([-\d.]+)/`.

## Outbound Message Contracts

### Thread Reply (Confirmation)

**When**: After successful pipeline processing of an inbox message.

```typescript
interface ConfirmationReply {
  msgtype: 'm.text';
  body: string;                // "Saved as **{category}**: {title}"
  'm.relates_to': {
    rel_type: 'm.thread';
    event_id: string;          // Original message event ID
    is_falling_back: true;
    'm.in_reply_to': { event_id: string };
  };
}
```

### Thread Reply (Clarification Request)

**When**: Pipeline Stage 1 determines clarification is needed.

```typescript
interface ClarificationReply {
  msgtype: 'm.text';
  body: string;                // "I need a bit more info:\n- Question 1\n- Question 2"
  'm.relates_to': {
    rel_type: 'm.thread';
    event_id: string;
    is_falling_back: true;
    'm.in_reply_to': { event_id: string };
  };
}
```

### Thread Reply (Error)

**When**: Pipeline processing fails.

```typescript
interface ErrorReply {
  msgtype: 'm.text';
  body: string;                // "Error processing message: {error.message}"
  'm.relates_to': {
    rel_type: 'm.thread';
    event_id: string;
    is_falling_back: true;
    'm.in_reply_to': { event_id: string };
  };
}
```

### Plain Message (Proximity Alert)

**When**: User's location is near a Place with pending items and cooldown has expired.

```typescript
interface ProximityAlertMessage {
  msgtype: 'm.text';
  body: string;
  // Template: "Hey! You're near {place.name} (~{distance}m). You have items on your list there:\n{itemLines}\nWorth a stop if you have time!"
  format: 'org.matrix.custom.html';
  formatted_body: string;
}
```

### Plain Message (Digest)

**When**: Daily or weekly cron fires and sends to the digest room.

```typescript
interface DigestMessage {
  msgtype: 'm.text';
  body: string;                // Markdown-formatted summary
  format: 'org.matrix.custom.html';
  formatted_body: string;      // HTML-formatted summary
}
```

### Plain Message (Wizard Prompt)

**When**: Bot sends prompts during the setup wizard flow.

```typescript
interface WizardMessage {
  msgtype: 'm.text';
  body: string;                // Wizard prompt text
  format: 'org.matrix.custom.html';
  formatted_body: string;
}
```

  content: string,                         // Raw message text

## LLM I/O Contracts

### Clarification Check

**Input** (system prompt): Built by `buildClarifyPrompt()`. Instructs the LLM to evaluate message ambiguity.
**Input** (user message): Built by `buildClarifyUserMessage(content)`. Format: `"Evaluate whether this captured message needs clarification before it can be processed:\n\n{content}"`

**Output** (Zod-validated `ClarifySchema`):
```typescript
{
  needsClarification: boolean;
  questions: string[];    // 1-3 clarifying questions, or empty array
  confidence: number;     // 0.0 to 1.0
}
```

### Classification + Field Extraction

**Input** (system prompt): Built by `buildClassifyPrompt(dbContext)`. Contains category definitions, field schemas per category, owner assignment rules, and current system context.
**Input** (user message): Built by `buildClassifyUserMessage(content, capturedBy)`. Format: `"Captured by: {capturedBy}\n\nMessage:\n{content}"`

**Output** (Zod-validated `ClassificationSchema`):
```typescript
{
  category: 'task' | 'project' | 'waiting_for' | 'event' | 'contact' |
            'resource' | 'note' | 'shopping' | 'someday_maybe' | 'area';
  confidence: number;           // 0.0 to 1.0
  needsClarification: boolean;
  clarifyingQuestions: string[];
  owner: 'alice' | 'bob' | 'shared';
  createdBy: string;
  fields: Record<string, unknown>;  // Category-specific fields
}
```

**Category-specific `fields` schemas**:

| Category | Fields |
|---|---|
| task | `title`, `description`, `priority`, `context`, `dueDate`, `projectName` |
| project | `name`, `description`, `outcome`, `nextAction`, `deadline` |
| waiting_for | `title`, `context`, `contactName`, `followUpDate` |
| event | `title`, `startAt`, `endAt`, `location`, `attendees` |
| contact | `name`, `relationship`, `company`, `email`, `phone`, `notes` |
| resource | `title`, `url`, `sourceType`, `author`, `keyTakeaways` |
| note | `title`, `content`, `tags` |
| shopping | `item`, `quantity`, `estimatedCost`, `whereToBuy`, `urgency` |
| someday_maybe | `title`, `description`, `category`, `reviewDate` |
| area | `name`, `description`, `successCriteria`, `reviewFrequency` |

### Enrichment

**Input** (system prompt): Built by `buildEnrichPrompt(category, fields, dbContext)`. Contains category-specific enrichment guidance and system context.
**Input** (user message): Built by `buildEnrichUserMessage(category, fields)`. Format: `"Enrich this {category}:\n{JSON.stringify(fields)}"`

**Output**: A flat JSON object with the same field names as the input, values enriched/completed. No wrapping schema -- parsed as `Record<string, unknown>`. If parsing fails, the original fields are used.

## Domain Events (Application-Level)

This section catalogs the logical events that flow through the system. These are not published to an external bus; they are internal cause-and-effect chains within the bot process.

| Event | Trigger | Producer | Consumer | Payload |
|---|---|---|---|---|
| InboxMessageReceived | m.text in inbox room | Matrix sync loop | InboxHandler | roomId, eventId, userId, text |
| ClarificationRequested | Pipeline Stage 1 needsClarification=true | AI Pipeline | InboxHandler (stores pending, sends thread reply) | userId, roomId, eventId, questions[] |
| ClarificationReceived | m.text from user with pending clarification | InboxHandler | AI Pipeline (re-enters at Stage 3) | originalMessage, clarificationText |
| RecordCreated | Pipeline Stage 5 writes to DB | AI Pipeline | InboxHandler (sends confirmation reply) | category, recordId, title, owner |
| ProjectRoomNeeded | Pipeline returns newProjectRoom=true | AI Pipeline | InboxHandler (creates room, updates DB) | projectName, members[], spaceId |
| ProjectRoomMessageReceived | m.text in project room | Matrix sync loop | ProjectHandler | roomId, eventId, userId, text, projectId |
| LocationReceived | m.location event | Matrix sync loop | LocationHandler | userId, roomId, lat, lon |
| ProximityAlertTriggered | User within ALERT_RADIUS_METERS of Place with pending items | Proximity engine | LocationHandler (sends message) | placeName, distance, items[] |
| DailyDigestTriggered | Cron fires at daily_digest schedule | node-cron | DailyCron (queries DB, sends message) | tasksDue[], eventsToday[], followUpsDue[] |
| WeeklyReviewTriggered | Cron fires at weekly_review schedule | node-cron | WeeklyCron (queries DB, sends message) | staleProjects[], overdueTasks[], overdueReviews[] |
| EnrichmentCronTriggered | Cron fires at enrichment schedule | node-cron | EnrichmentCron (queries DB, logs stale items) | staleInboxItemIds[] |
| SetupWizardStarted | Admin sends "!setup" | WizardHandler | Bot (sends wizard prompts) | userId, roomId |
| SetupWizardCompleted | Admin finishes wizard | WizardHandler | Space/room creator, config writer | spaceName, spaceId, rooms, users |
